### Deployment Attempts — What Failed
1. **Hostinger Docker Compose from URL** — "Docker project not found" error. Hostinger's system constructs invalid GitHub URLs (`github.com/user/repo/file` instead of `raw.githubusercontent.com/user/repo/branch/file`). This is a Hostinger platform bug.
2. **Hostinger Compose from URL with raw URL** — Same error.
3. **SSH via Kodee (Hostinger's built-in AI)** — Git clone only pulled `.env` file (12 bytes), not the actual source. The repo was cloned but source files weren't there.
4. **Multiple GitHub PATs** — Several tokens were used/revoked. The current working token was used to push successfully.
